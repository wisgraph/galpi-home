### 전략 1. "The 100-Link Trap" (강력 추천 🏆)

모든 기능을 100% 열어주되, 저장할 수 있는 **링크의 개수만 제한**하는 방식입니다.

- **방식:** "무료 버전에서는 최대 **50개(또는 100개)**의 링크를 저장할 수 있습니다."
- **심리:**
    1. 사용자는 모든 기능(딥링크, 태깅, 검색)을 경험하며 "와, 이거 진짜 편하네"라고 느낍니다.
    2. 링크가 50개가 쌓일 때쯤이면 이미 이 앱은 **업무 프로세스의 일부**가 되어 있습니다.
    3. 51번째 링크를 저장하려는 순간, **"단돈 $4.99로 무제한 해방되세요"** 팝업이 뜨면 결제 저항감이 '0'에 수렴합니다.
- **구현:** Rust 코드에서 `data.jsonl`의 라인 수(Line Count)만 체크하면 되니 구현도 가장 쉽습니다.

### 전략 2. "Viewer is Free, Creator pays" (신뢰 구축용)

이것은 **"데이터 주권"** 을 강조하는 선생님의 앱 철학과 완벽하게 맞아떨어지는 전략입니다.
- **방식:**
    - **검색 & 열기 (Read):** **평생 무료.** 라이선스가 없어도 저장된 링크를 검색하고 실행하는 건 무제한입니다.
    - **새 링크 추가/수정 (Write):** 유료 기능.
- **마케팅 포인트:**
    > _"우리는 당신의 데이터를 인질로 잡지 않습니다. 나중에 HookLink 사용을 중단하더라도, 당신이 저장한 링크는 평생 무료로 검색하고 열 수 있습니다. 안심하고 시작하세요."_
- **효과:** 사용자가 "나중에 유료 결제 안 하면 내 자료 못 보게 되는 거 아냐?"라는 **두려움(Risk)** 을 원천 차단해 줍니다. 훅마크나 구독형 앱들이 절대 따라 할 수 없는 강점입니다.

### 전략 3. "Time-Limited Trial" (14일 무료)

- **방식:** 설치 후 14일간 풀 버전 제공, 그 후엔 잠금.
- **비추천 이유:** 선생님의 앱은 '도구'입니다. 도구는 손에 익을 시간이 필요합니다. 바쁜 직장인은 설치만 해두고 14일 동안 못 써볼 수도 있습니다. 그러면 그냥 지워버립니다. '개수 제한'이 낫습니다.

### 🛠️ 기술적 구현 가이드 (Rust)

라이선스 체크 로직을 `JSONL` 저장 시점에만 넣으면 됩니다.

```rust
// 가상의 Rust 로직 (개념)
fn save_link(new_link: Link) -> Result<(), Error> {
    let current_count = count_lines_in_jsonl()?; // 현재 저장된 개수 확인
    let has_license = check_license_key(); // 라이선스 보유 여부 확인 (로컬 파일 체크)

    // 무료 사용자이면서 50개가 넘었으면 저장 차단
    if !has_license && current_count >= 50 {
        return Err(Error::LicenseLimitReached); 
        // 이때 프론트엔드에서 "업그레이드 하세요!" 모달을 띄움
    }

    append_to_jsonl(new_link)?;
    Ok(())
}
```

결론: 무료 기능을 넣되, "기능을 쪼개지 말고(No Feature Cut), 양을 제한하세요(Cap Quantity)."

그리고 **"읽기는 평생 무료"** 라고 선언하세요. 이것이 훅마크를 이기는 가장 확실한 '혜자 앱' 전략입니다.

이제 진짜 준비가 다 된 것 같습니다.

1. **백엔드:** Rust + JSONL + In-Memory (동기화/충돌 방지 로직 포함)
2. **안전장치:** BookmarkData + xattr (링크 유실 방지)
3. **결제:** Lemon Squeezy ($4.99 얼리버드)
4. **무료 정책:** 50개 제한 + 읽기 전용 무료
