### 1. 핵심 페인포인트 (Key Pain Points)
_우선순위: 높음 → 낮음_
1. **구독 피로감 (Subscription Fatigue):**
    - 단순 유틸리티인 훅마크(Hookmark)가 연간 약 $70(9만원)이라는 고가 구독 모델을 고수함에 따라 발생하는 강력한 심리적 저항감. 사용자는 "기능은 좋은데, 이걸 매달 돈 내고 써야 해?"라는 의문을 가짐1111.
2. **데이터 락인(Lock-in) 공포:**
    - 훅마크는 자체적인 독점 데이터베이스(DB)에 연결 정보를 저장함. 사용자는 구독을 끊거나 앱이 사라지면 힘들게 연결해둔 지식 네트워크가 증발할 수 있다는 불안감을 느낌2222.
3. **앱의 무거움과 퍼포먼스 저하:**
    - 단순한 링크 복사/이동을 원하는데, 훅마크는 너무 많은 메타데이터를 관리하느라 무겁고 느려짐(Lag). 0.1초의 지연도 용납하지 않는 런처 사용자에게는 치명적임333333333.
4. **한국어 입력 환경의 마찰:**
    - Raycast나 Alfred 사용 시, 한글 자소 분리(NFD) 문제나 한영 전환 실수(Typo)로 인해 검색 흐름이 끊김. "이력서"를 찾으려는데 영어로 타이핑되거나, 파일명이 깨져 보이는 경험4444.
5. **컨텍스트 스위칭 비용:**
    - 작업 중 참고 자료(PDF, 웹사이트)를 찾기 위해 Finder나 브라우저를 뒤적거리는 동안 원래 하려던 생각의 흐름(Flow)이 깨짐. "내가 뭘 찾으려고 했더라?"라는 인지적 부하 발생5555.

### 2. 주요 니즈 (Key Needs)
_우선순위: 높음 → 낮음_
1. **속도와 즉시성 (Speed & Immediacy):**
    - 무거운 DB 로딩 없이, 단축키를 누르는 순간 즉시 링크가 생성되고 검색창이 떠야 함. "기다리지 않는 경험"이 최우선 가치.
2. **데이터 주권 (Plain Text Compatibility):**
    - 앱이 없어져도 링크는 살아있어야 함. 옵시디언(Obsidian)이나 노션(Notion)에 붙여넣은 마크다운 링크(`[제목](hooklink://...)`)가 영구적으로 작동하기를 원함.
3. **합리적인 가격 (One-Time Purchase):**
    - 매달 나가는 고정비가 아니라, 한 번 구매하면 평생 내 것이 되는 '소유권'을 원함. (예: 2~3만 원대 일회성 구매) .
4. **한국어 검색 최적화:**
    - '네이버'를 'ㄴㅇㅂ'로 검색해도 찾아주는 초성 검색과, 한영 전환을 신경 쓰지 않아도 알아서 처리해 주는 똑똑한 입력창을 원함.
5. **가벼운 연결 (Lightweight Linking):**
    - 복잡한 양방향 그래프 시각화보다는, 단순히 "지금 보고 있는 걸 복사해서 저기에 붙여넣기"하는 빠르고 단순한 워크플로우를 선호함.

### 4. 기회 영역 (Business/User Value)

- **Raycast의 틈새 공략 (The "Extension" Gap):**
    - Raycast는 링크를 _소비_하는 데에는 좋지만, 딥링크를 _생성_하는 기능은 약함. Hooklink는 Raycast와 경쟁하는 것이 아니라, Raycast 사용자가 "복사" 기능을 위해 반드시 설치해야 하는 **필수 백엔드 유틸리티**로 포지셔닝111111111111111111.
- **한국형 PKM 시장 선점:**
    - 옵시디언, 노션 등을 사용하는 한국의 '지식 근로자'들은 늘어나고 있으나, 이들을 위한 한글 최적화 툴은 전무함. 초성 검색과 자소 분리 해결은 글로벌 앱이 따라올 수 없는 강력한 해자(Moat)가 됨12121212.
- **"가벼움"이라는 프리미엄:**
    - 기능이 많아서 좋은 게 아니라, "없어서(DB가 없어서, 구독이 없어서)" 더 좋다는 역발상 마케팅. Rust/Tauri 기반의 가벼움 자체가 세일즈 포인트131313131313131313.

### 5. 추천 기능 (Recommended Features)

_페인포인트를 해결하는 구체적 기능_

1. **Instant Capture (DB-Less):**
    - 별도 저장 과정 없이, 단축키(`Cmd+Shift+C`)만 누르면 현재 창의 딥링크를 마크다운/Rich Text 포맷으로 클립보드에 즉시 복사. (본문 저장 X, 오직 링크만).
2. **스마트 한글 입력기 (Context-Aware Input):**
    - 검색창 호출 시 자동으로 한글/영문 입력 모드를 보정. 검색어 입력 중 오타 발생 시(예: `dlfurd` -> `이력서`) 자동으로 변환하여 검색.
3. **초성 검색 (Choseong Search):**
    - 저장된 링크나 히스토리를 검색할 때 'ㄱㅂ'만 입력해도 '공부.md'나 '개발문서'를 찾아주는 한국어 네이티브 검색 알고리즘 탑재16.
4. **Local History (휘발성 트레이):**
    - 영구 DB 대신, 최근 생성한 링크 50~100개만 보여주는 가벼운 '히스토리 트레이' 제공. "아까 복사했던 그 링크"를 다시 찾을 때 유용하며, 앱 종료 시 삭제 옵션 제공17.
5. **Markdown "Paste As" Options:**
    - 단순 URL뿐만 아니라, 옵시디언 사용자를 위한 `[Title](hook://...)` 포맷, 노션 사용자를 위한 임베드 포맷 등을 설정에서 선택 가능하도록 지원18.

# 1. Executive Summary (제안 요약)

## Key Message:
"Hookmark의 무거움과 구독료를 덜어낸, 전 세계에서 가장 빠르고 가벼운 'Stateless' 딥링크 런처"

**Problem:**
- **Bloatware & Lag:** 기존 시장 지배자인 Hookmark는 단순한 링크 연결을 위해 무거운 데이터베이스를 구동하여 시스템 리소스를 점유하고 반응 속도가 느림111.
- **Subscription Fatigue:** 유틸리티 앱임에도 불구하고 연간 약 $70의 고가 구독 모델을 고수하여 사용자들의 피로감과 이탈률이 높음2.
- **Data Lock-in:** 연결 정보가 독점 DB에 저장되어, 앱 사용을 중단하면 사용자의 지식 네트워크가 단절될 위험이 있음3.

**Solution:**
- **Rust + Tauri Engine:** Electron 대비 메모리 사용량을 획기적으로 줄이고(1/10 수준), 0.1초 이내의 즉각적인 반응 속도를 보장하는 초경량 아키텍처 구현444.
- **Stateless Connectivity:** 별도의 DB 저장 없이 링크를 생성하여 클립보드로 전달하는 '무상태(Stateless)' 방식을 채택, 데이터 주권을 100% 사용자에게 부여5555.
- **Universal Fuzzy Finder:** 영문뿐만 아니라 CJK(한중일) 유니코드 환경까지 완벽하게 지원하는 고성능 퍼지 검색 엔진 탑재.

**Goal:**
- **Market Validation:** 글로벌 PKM(Personal Knowledge Management) 커뮤니티(Obsidian, DEVONthink 등)에서 "Hookmark의 합리적 대안"으로 인정받아 **유료 사용자 1,000명** 확보.
- **Sustainable Revenue:** **$29 평생 라이선스(Lifetime License)** 모델을 통해 1인 개발자의 지속 가능한 수익 구조(Cash Cow) 창출6666.

기존 시장(Hookmark, Raycast 사용자 커뮤니티)에서 수집된 주요 불만 사항은 다음과 같습니다.

- **"내 데이터가 인질로 잡혀있다."** : 앱을 구독 해지하면 연결해둔 링크들이 사라질까 봐 두려워하는 '데이터 종속성' 공포15.

## 2.3. 문제 정의 및 목표 설정 (Problem & Objectives)

### 2.3.1. 핵심 문제 도출 (Root Cause)

- **과잉 엔지니어링 (Over-Engineering):** 기존 솔루션(Hookmark)은 단순한 '연결'을 위해 자체 DB를 구축하는 등 과도하게 무거운 구조를 택하여 퍼포먼스 저하를 유발합니다17.
    
- **로컬라이제이션의 공백:** 글로벌 런처들은 한국어라는 특수한 입력 환경(IME)과 파일 시스템 환경(NFD/NFC)을 고려하지 않아, 한국 사용자들에게 미세하지만 반복적인 '마찰(Friction)'을 일으킵니다.
    

### 2.3.2. 기획 목적 (Purpose)

- **Lightweight & Stateless:** 자체 DB 없는 '무상태(Stateless)' 아키텍처를 통해, 시스템 리소스를 점유하지 않는 **가장 가볍고 빠른 딥링크 도구**를 제공합니다18.
    
- **Native Korean Experience:** 초성 검색, 자소 분리 자동 보정, 문맥 인식 입력기 등을 통해 한국 맥 사용자에게 **'마찰 없는(Frictionless)' 연결 경험**을 제공합니다19.
    
- **Data Sovereignty:** 사용자의 데이터를 앱에 가두지 않고, 표준 프로토콜(Markdown, URL Scheme)을 활용하여 **데이터 주권**을 사용자에게 돌려줍니다20.
    

### 2.3.3. 목표 (Goal)

거창한 매출 목표보다는, 1인 개발자로서 지속 가능한 **'니치 마켓 안착'**을 최우선으로 합니다.

1. **초기 팬덤 확보:** 한국의 생산성 도구(Obsidian, Notion) 커뮤니티를 중심으로 **유료 구매 전환 사용자 1,000명** 확보 (초기 PMF 검증).
    
2. **시장 포지셔닝:** "Hookmark의 가벼운 한국형 대안"이라는 명확한 인식을 심어주어, 한국 맥 앱스토어 생산성 카테고리 내 **유틸리티 필수 앱**으로 진입.
    
3. **수익 모델 검증:** 구독이 아닌 **'합리적 평생 라이선스(One-time Purchase)'** 모델이 한국 시장에서 통한다는 것을 증명21.
    

# 3. Concept & Strategy (전략 및 컨셉)

## 3.1. 타겟 오디언스 (Target Audience)

### 3.1.1. 메인 타겟 (Core Target)

**"구독 피로에 지친 로컬 PKM(Personal Knowledge Management) 헤비 유저"**

# 4. Detailed Plan (구체적 실행 방안)

본 프로젝트는 **Tauri v2 + Svelte 5** 기반의 데스크탑 애플리케이션으로, **로컬 우선(Local-First)** 아키텍처와 **키보드 중심(Keyboard-Centric)** UX를 최우선으로 설계합니다.

## 4.1. 서비스 구조 (Architecture)

### 4.1.1. IA (Information Architecture)
앱은 단일 윈도우(Launcher Window) 내에서 상태에 따라 뷰를 스위칭하는 **SPA(Single Page Application) 구조**를 가집니다.

*   **Launcher (Root Container)**
    *   **Header:** `SearchBar` (검색어 입력, 태그 자동완성 Trigger)
    *   **Body (Router View)**
        *   `SEARCH` (Default): 검색 결과 리스트 (`HookCard` List)
        *   `CAPTURE`: 현재 컨텍스트 캡처 프리뷰 (`HookCaptureCard`)
        *   `SETTINGS`: 설정 패널 (`General`, `Shortcuts`)
        *   `HELP`: 단축키 치트 시트 (`Grid Layout`)
        *   `CONTEXT_MENU`: 액션 리스트 오버레이
    *   **Footer:** 상태 표시줄 (Action Hints, Toast Message)

### 4.1.2. User Flow (주요 동선)

1.  **Capture Flow (저장)**
    *   User: 글로벌 단축키 입력 (`Option+Space`)
    *   System: 활성 윈도우(Active Window) 감지 -> 정보 추출 -> `CAPTURE` 뷰 전환
    *   User: 제목 수정 or 태그(`#`) 입력 -> `Enter`
    *   System: DB 저장 -> 윈도우 숨김 -> `SEARCH` 뷰 리셋

2.  **Search & Execute Flow (실행)**
    *   User: 글로벌 단축키 입력 (`Option+Space`)
    *   System: `SEARCH` 뷰 렌더링 (최근 항목 표시)
    *   User: 키워드(초성/텍스트) 입력 -> `ArrowUp/Down` 이동 -> `Enter`
    *   System: 해당 `hook.address` 실행 (브라우저/앱 열기) -> 사용 횟수 증가 -> 윈도우 숨김

3.  **Context Action Flow (관리)**
    *   User: 리스트에서 항목 선택 -> `Cmd+K` (또는 우클릭)
    *   System: `CONTEXT_MENU` 오버레이 호출
    *   User: 액션 선택 (Copy, Edit, Delete) -> 실행

## 4.3. 기술 및 운영 전략 (Tech & Operation)

### 4.3.1. 시스템 아키텍처 (System Architecture)
*   **Frontend (UI Layer):**
    *   **Stack:** Svelte 5 (Runes), TypeScript, Tailwind CSS.
    *   **Pattern:** MVVM 변형 (View -> Controller -> Service/Store).
    *   **State:** 전역 반응형 객체(`$state`)를 통한 단일 진실 공급원(Single Source of Truth) 유지.
*   **Backend (Core Layer):**
    *   **Stack:** Rust (Tauri v2).
    *   **Role:** OS API 호출(윈도우 제어, 클립보드, 파일시스템), SQLite DB 트랜잭션 관리.
    *   **Performance:** 한글 자모 분해 및 무거운 연산의 오프로딩.
*   **Communication (IPC):**
    *   Tauri `invoke` (Frontend -> Backend 요청).
    *   Tauri `emit` (Backend -> Frontend 이벤트, 예: `preview-hook`).

### 4.3.2. 데이터 관리 및 운영 (Operation)
*   **Local-First Policy:** 모든 데이터는 사용자 로컬 기기의 SQLite (`hook.db`)에 저장됩니다. 별도의 서버 비용이 발생하지 않으며, 오프라인에서도 완벽하게 동작합니다.
*   **Virtual Protocol:** `virtual://settings`, `virtual://help` 등의 내부 프로토콜을 사용하여, 외부 URL 실행 로직과 내부 페이지 이동 로직을 통일성 있게 관리합니다.
*   **Update Strategy:** Tauri의 내장 업데이터를 활용하여 GitHub Releases와 연동된 자동 업데이트를 제공합니다.

### 4.3.3. 마케팅 및 배포 (Go-to-Market)
*   **Target Persona:** 개발자, 리서처, 키보드 헤비 유저.
*   **Key Message:** "Context Switching Cost를 0으로 만드세요."
*   **Channels:**
    *   GitHub Open Source (개발자 커뮤니티 신뢰 확보).
    *   Product Hunt 런칭.
    *   Reddit (r/macapps, r/productivity) 바이럴 마케팅.
