# Hooklink: Context Tag System
> **작업의 맥락(Context)을 고정하고, 정보의 흐름을 가속화하세요.**

Hooklink의 **컨텍스트 태그(Context Tag)** 시스템은 사용자가 현재 집중하고 있는 특정 프로젝트나 작업 환경을 앱 전역에 '고정'하는 기능입니다. 단순한 분류를 넘어, 새로운 정보를 수집하고 기존 정보를 필터링하는 모든 과정을 자동화합니다.

[영상: 컨텍스트 태그가 활성화된 상태에서 여러 웹페이지를 빠르게 캡처할 때, 별도의 입력 없이 모든 캡처 카드에 해당 태그가 자동으로 박히는 모습]

---

## 1. 컨텍스트 태그란 무엇인가요?

일반적인 태그가 과거의 기록을 분류하기 위한 수단이라면, **컨텍스트 태그는 '현재와 미래'를 위한 선언**입니다. 특정 태그(예: `#project:antigravity`)를 컨텍스트로 활성화하면, Hooklink는 사용자가 현재 해당 프로젝트 모드에 있음을 인지하고 환경을 최적화합니다.

[스크린샷: 검색창 우측에 선명한 컬러의 알약 모양으로 고정된 '활성화된 컨텍스트 태그' UI]

### 핵심 매커니즘
- **자동 주입 (Auto-Injection):** 컨텍스트가 활성화된 상태에서 새로운 웹페이지, 문서, 앱을 캡처하면 해당 태그가 자동으로 부여됩니다.
- **포커스 필터링 (Focus Filtering):** 활성화된 태그를 기반으로 관련 검색 결과를 우선적으로 제시하여 작업 환경 구성을 돕습니다.
- **즉시 할당 (Instant Assignment):** 리스트를 훑으며 단축키 하나로 기존 자료들을 현재 컨텍스트에 편입시킬 수 있습니다.

---

## 2. 주요 기능 및 사용법

### 🎯 컨텍스트 등록하기
1. **검색창 활용:** 검색창에서 `#`를 입력해 태그 자동완성을 띄운 뒤, `Cmd + Enter`를 누르면 해당 태그가 우측 상단 컨텍스트 바에 고정됩니다.
2. **기존 데이터 활용:** 훅 리스트에 이미 달려있는 태그를 **더블 클릭**하여 즉시 컨텍스트로 승격시킬 수 있습니다.

[영상: 검색창 자동완성에서 Cmd+Enter로 컨텍스트를 등록하고, 이어서 리스트의 태그를 더블클릭하여 컨텍스트가 교체되는 인터랙션]

### ⚡️ 쾌속 작업 (Fast Workflow)
- **태그 자동 추가:** 새로운 컨텍스트를 캡처할 때 일일이 태그를 입력할 필요가 없습니다. 설정된 컨텍스트가 자동으로 따라붙습니다.
- **리스트에서 바로 추가:** 컨텍스트 태그가 켜져 있을 때, 리스트의 특정 항목 위에서 **`Cmd + Shift + A`**를 누르면 모든 활성 컨텍스트 태그가 즉시 해당 항목에 할당됩니다.
- **고스트 버튼 활용:** 마우스 호버 시 나타나는 **`+ [태그명]`** 버튼을 클릭하여 선택적인 할당도 가능합니다.
- **일괄 제거 (Clear All):** 작업 맥락이 바뀌었나요? **`Cmd + Esc`**를 누르면 등록된 모든 컨텍스트 태그가 즉시 제거되어 깨끗한 상태로 돌아갑니다.
- **개별 제거:** 태그를 잘못 달았나요? 컨텍스트 바의 태그 알약을 클릭하거나, 항목의 태그 위에서 **우클릭** 한 번이면 즉시 제거됩니다.

[스크린샷: 훅 카드에 마우스를 올렸을 때 태그 리스트 끝에 나타나는 투명한 '+컨텍스트태그' 고스트 버튼]

## 3. 단축키 요약 (Shortcuts Guide)

| 기능 | 단축키 | 위치 |
| :--- | :--- | :--- |
| **컨텍스트 등록** | `Cmd + Enter` | 태그 자동완성 창 |
| **컨텍스트 승격** | `Double Click` | 리스트 내 태그 알약 |
| **컨텍스트 일괄 삭제** | **`Cmd + Esc`** | 앱 전역 (검색/캡처) |
| **현재 항목에 맥락 추가** | `Cmd + Shift + A` | 리스트 선택 항목 |
| **태그 개별 삭제** | `Mouse Click` | 상단 컨텍스트 바 |

## 4. 활용 사례 (Use Cases)

### Case A: 실시간 리서치 및 논문 작성
> "다양한 논문과 웹 자료를 읽으며 특정 주제에 대해 조사 중일 때"
- `#research:ai_ethics` 태그를 컨텍스트로 고정합니다.
- 브라우저에서 읽고 있는 모든 중요 페이지를 Hooklink로 캡처만 하세요. 모든 자료에 자동으로 태그가 붙어 나중에 한 번에 모아볼 수 있습니다.

### Case B: 멀티 프로젝트 스위칭
> "오전에는 'A 사이트 리뉴얼' 작업을 하고, 오후에는 'B 앱 기획'을 해야 할 때"
- 작업 전환 시 컨텍스트 태그만 교체하세요. 
- 검색창에 `#project:site_a`가 고정되는 순간, 해당 프로젝트와 관련된 피그마 시안, 노션 기획서, 참고 레퍼런스들만 필터링되어 나타납니다. 작업 환경을 복구하는 데 드는 시간이 0초로 단축됩니다.

[스크린샷: 컨텍스트 태그가 바뀜에 따라 검색 결과 리스트가 해당 프로젝트 중심으로 역동적으로 필터링된 화면]

### Case C: 자료 아카이빙 및 정리
> "기존에 수집했던 수백 개의 링크를 특정 카테고리로 빠르게 정리해야 할 때"
- 정리할 목적지 태그(예: `#read_later`)를 컨텍스트로 등록합니다.
- 전체 리스트를 아래로 내리며 단축키(`Cmd+Shift+A`)를 툭툭 누르기만 하세요. 마우스 클릭 없이도 수십 개의 링크를 1분 안에 분류할 수 있습니다.

[영상: 훅 리스트를 키보드 화살표로 내려가며 Cmd+Shift+A를 연타하여, 순식간에 여러 항목에 태그가 촤르륵 추가되는 모습]

---

## 4. 철학: "정리하기 위해 멈추지 마세요"

Hooklink의 컨텍스트 태그 시스템은 사용자의 흐름(Flow)을 방해하지 않는 것을 최우선으로 합니다. 정보를 저장할 때마다 "어디에 분류할지" 고민하고 입력하는 인지적 비용을 제거하여, 작성과 실행에만 집중할 수 있는 환경을 제공합니다.
