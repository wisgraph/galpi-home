# Hooklink: High-Performance Search Engine
> **0.1초의 망설임 없는 검색, 당신의 생각보다 빠른 응답.**

Hooklink의 검색 엔진은 단순히 텍스트를 대조하는 수준을 넘어선 고성능 검색 인프라입니다. Rust 기반의 강력한 엔진과 한국어 환경에 최적화된 알고리즘을 통해 수만 개의 훅 사이에서도 원하는 정보를 즉각적으로 찾아냅니다.

---

## 1. Rust 기반 Fuzzy Finding 기술

Hooklink의 백엔드는 고성능 검색 라이브러리인 **Nucleo-matcher**를 Rust 환경에서 운용합니다. 이는 현대적인 IDE나 완성형 검색 도구에서 사용하는 최고 수준의 퍼지 매칭 알고리즘입니다.

- **유연한 대조:** 사용자가 단어의 순서를 바꾸거나 일부 철자를 틀려도 검색 의도를 파악하여 관련 결과를 보여줍니다.
- **지능적 스코어링 (Scoring):** 일치하는 위치, 길이, 그리고 사용자의 최근 사용 빈도(Mouse Weight)를 종합적으로 계산하여 가장 확률이 높은 항목을 상단에 배치합니다.

[영상: 검색창에 오타를 섞어 입력하거나 아주 일부분만 입력해도 원하는 결과가 실시간으로 최상단에 뜨는 장면]

---

## 2. 한국어 최적화: 초성 검색 (Chosung Search)

한국어 사용자에게 가장 직관적인 검색 방식인 **초성 검색**을 완벽하게 지원합니다. 단어의 자음만 입력해도 즉각적인 필터링이 가능합니다.

- **예시:** `ㄷㅅㅌㄹ` 입력 시 `#독서토론` 태그가 즉시 검색됩니다.
- **구현 원리:** 검색 요청 시 한글 음절을 초성/중성/종성으로 실시간 분해하여 매칭 인덱스를 대조하며, 일반 한글 입력과 초성 입력을 동시에 지원합니다.

[영상: 검색창에 'ㄱㄴ'만 쳤을 때 '가나다', '강남', '기획' 등이 순식간에 필터링되는 모습]

---

## 3. 밀리초 단위의 응답성 (Latency-free)

Hooklink의 검색 인프라는 인덱싱 방식과 실시간 매칭 방식의 장점을 결합하여 설계되었습니다.

- **인메모리 엔진:** 데이터가 늘어나도 검색 성능이 저하되지 않도록 최적화된 데이터 구조를 메모리 상에서 운용합니다.
- **논블로킹 인터페이스:** 사용자가 타이핑하는 동안 메인 UI 스레드가 멈추지 않으며, 모든 검색 연산은 백그라운드에서 병렬로 처리됩니다.

---

## 4. 검색 경험의 완성: 태그와 텍스트의 결합

단순 제목 검색뿐만 아니라 태그, 주소, 앱 이름까지 한 번에 통합 검색이 가능합니다.

- **태그 가중치:** `#`로 시작하는 태그 검색 시 검색 엔진은 이를 핵심 맥락으로 인지하여 가중치를 높게 설정합니다.
- **글로벌 필드 매칭:** 제목뿐만 아니라 앱 종류(`Chrome`, `Figma` 등)나 원문 주소를 검색하더라도 퍼지 파인더가 이를 감지하여 제안합니다.

[스크린샷: 검색창에 한 두 글자만 입력했음에도 제목, 앱 이름, 태그가 복합적으로 고려되어 정렬된 고해상도 검색 결과 리스트]

---

**Hooklink의 검색은 기다림이 없습니다. 당신이 검색어를 완성하기도 전에 이미 답은 리스트에 올라와 있을 것입니다.**
