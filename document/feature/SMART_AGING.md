# 쓰레기 링크(Link Clutter) 자동 청소: 스마트 에이징 기술

링크를 수집하다 보면 필연적으로 마주치는 문제가 있습니다.
1,000개, 5,000개... 데이터가 쌓일수록 검색 결과는 지저분해집니다.
1년 전에 끝난 프로젝트 파일이 가장 위에 뜨고, 어제 작업한 중요한 파일은 저 아래 묻혀버립니다.
이것이 정보 관리 앱을 쓰다가 포기하게 만드는 주범, **"링크 쓰레기장(Link Clutter)"** 현상입니다.

HookLink는 이 문제를 해결하기 위해 검색 엔진에 **"시간의 개념"**을 도입했습니다.

---

## 1. 죽은 링크가 검색을 방해합니다

기존 앱(Hookmark 등)은 모든 링크를 평등하게 대우합니다.
3년 전 파일과 오늘 만든 파일이 텍스트만 같으면 똑같은 점수를 받습니다.
하지만 사용자의 뇌는 그렇지 않습니다. **"최근에 쓴 것"이 압도적으로 더 중요합니다.**

HookLink는 인간의 기억 방식과 동일하게 작동하도록 설계되었습니다.

## 2. 해결책: 스마트 에이징 (Smart Aging) 알고리즘

우리는 검색 랭킹 시스템에 정교한 **시간 감쇠(Recency Decay)** 공식을 적용했습니다.
별도의 '아카이브' 버튼을 누르거, 폴더를 정리할 필요가 없습니다. 그냥 쓰면 됩니다.

### 작동 메커니즘
모든 링크는 생성된 순간부터 나이를 먹습니다.
$$ \text{Final Score} = \text{Relevance} \times \text{Time Factor} $$

*   **⚡ 초신성 부스트 (Recency Boost):**
    *   최근 **24시간 이내**에 사용한 링크는 검색 점수에 **1.5배** 가산점을 받습니다.
    *   키워드가 조금 부정확해도, 어제 쓴 파일이라면 최상단에 꽂힙니다.
*   **🍂 자연스러운 퇴적 (Natural Decay):**
    *   **6개월 이상** 열어보지 않은 링크는 점수가 **30% 수준**으로 낮아집니다.
    *   삭제되진 않지만, 검색 결과의 바닥으로 조용히 가라앉습니다.

## 3. 사용자 경험의 변화

이 기술이 적용되면, 런처를 사용할 때 **"항상 빠릿빠릿하고 신선하다"**는 느낌을 받게 됩니다.

*   **자동 청소:** 1년 전 자료가 검색창을 어지럽히지 않습니다.
*   **즉각적인 접근:** 방금 닫은 그 파일을 다시 열고 싶을 때, 단축키 한 번이면 100% 확률로 맨 위에 있습니다.
*   **신뢰도 상승:** 검색 결과가 내 의도와 정확히 맞아떨어지므로, 도구에 대한 신뢰가 쌓입니다.

## 요약

| 기능 | 기존 런처 (Spotlight, Hookmark) | HookLink |
| :--- | :--- | :--- |
| **랭킹 기준** | 텍스트 정확도 (Text Match) | **텍스트 + 시간 (Text + Time)** |
| **오래된 파일** | 최상단 노출 가능 (방해됨) | **검색 결과 하단으로 침전** |
| **최근 파일** | 다른 파일과 동일 취급 | **강력한 우선순위 부스트** |
| **관리 방법** | 수동으로 삭제/이동 필요 | **자동 에이징 (Zero Maintenance)** |

**HookLink는 당신의 기억처럼 작동합니다. 중요한 것은 선명하게, 오래된 것은 흐릿하게.**
