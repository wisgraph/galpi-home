# Hooklink: Auto-Context Discovery
> **캡처하는 순간, 당신의 모든 관련 지식이 한자리에 모입니다.**

Hooklink의 **자동 맥락 발견(Auto-Context Discovery)** 기술은 사용자가 정보를 저장하는 행위를 단순한 "기록"에서 "연결"로 격상시킵니다. 이미 저장된 정보를 다시 마주했을 때, 사용자가 요청하기 전에 시스템이 먼저 관련 맥락을 제안합니다.

---

## 1. 지능형 맥락 인지 엔진 (Context-Aware Engine)

이 기능의 핵심은 사용자의 현재 작업 상태와 과거의 지식 구조를 실시간으로 교차 검증하는 기술에 있습니다.

- **실시간 중복 감지:** 사용자가 특정 주소(URL, 파일 경로 등)를 캡처하는 즉시, 인메모리(JSONL-Sync) 엔진이 밀리초 단위로 기존 데이터베이스와 대조합니다.
- **메인 컨텍스트 태그 추론:** 감지된 중복 항목이 `@project`, `@client`, `@topic` 등 사전 정의된 메인 컨텍스트 태그를 포함하고 있다면, 엔진은 이를 현재 사용자의 최우선 관심사로 즉각 확정합니다.

---

## 2. 하이브리드 인터페이스 (Hybrid Capture-Search View)

정보를 캡처했을 때 단순히 "이미 저장됨"을 알리는 데 그치지 않고, 검색 리스트와 캡처 카드를 결합한 독창적인 UI를 제공합니다.

- **듀얼 레이어 뷰:** 화면 상단에는 현재 보고 있는 훅의 상세 정보(Capture Card)를 고정하고, 하단에는 해당 훅과 맥락을 공유하는 모든 항목을 검색 결과로 즉시 나열합니다.
- **자동 쿼리 주입(Auto-Query Injection):** 사용자가 타이핑하지 않아도 시스템이 내부적으로 `#태그` 검색 쿼리를 실행하여, 관련 정보를 찾는 데 드는 인지적 비용을 제로(0)로 만듭니다.

[영상: 이미 저장된 노션 페이지를 캡처하자마자, 하단 리스트에 해당 프로젝트와 관련된 피그마 시안, 슬랙 대화록, 관련 파일들이 주르륵 브리핑되듯 나타나는 모습]

---

## 3. 베스트 유즈케이스: 작업의 연속성 확보

### A. 프로젝트 온보딩 및 복귀
오랫동안 손을 놓았던 프로젝트의 파일을 하나 열고 캡처 단축키를 누르세요. Hooklink가 즉시 해당 프로젝트 태그로 검색 결과를 필터링하여, 함께 작업했던 모든 웹사이트와 참고 문서들을 리스트업해 줍니다. 당신은 단 1초 만에 과거의 작업 맥락으로 완벽하게 복귀할 수 있습니다.

### B. 지식의 입체적 연결
특정 논문이나 레퍼런스를 다시 읽을 때 캡처를 실행하면, 과거에 동일한 주제로 태깅해 두었던 다른 자료들이 하단에 자동으로 나타납니다. "아, 맞다! 이것도 있었지" 하는 유레카 모먼트를 시스템이 직접 만들어냅니다.

---

## 4. 기술적 완성도: 제로 레이턴시 데이터 매칭

- **표준화된 주소 매칭:** `file://` 프로토콜, 퍼센트 인코딩된 경로, 브라우저 탭 주소 등 파편화된 주소 체계를 단일 정규화 포맷으로 변환하여 완벽한 매칭을 보장합니다.
- **논블로킹 검색:** 맥락 감지와 검색 결과 도출 과정이 비동기로 처리되어, 캡처 화면이 뜨는 속도에 전혀 영향을 주지 않으면서도 검색 결과는 부드럽게 페이드인됩니다.

---

**Hooklink는 기억을 보조하는 도구를 넘어, 당신의 지식을 유기적으로 연결하는 뇌의 확장판이 됩니다.**
